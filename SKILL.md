# AI 漫剧导演

基于 neta 能力的全流程漫剧制作技能。核心流程：**剧情分析 → 镜头规划 → 分镜图生成 → 视频生成 → 视频合并**。

> **前置要求：** 需要 neta 技能可用

---

## 触发条件

用户说以下任意内容时触发本技能：
- 制作漫剧 / 做一集漫剧 / 生成漫剧
- 把这个故事/剧情做成视频
- AI 短剧 / 竖屏漫画动态化
- 分镜规划 + 视频生成

---

## Step 1：剧情摄入与分析

从用户输入中提取以下要素：

| 要素 | 说明 |
|------|------|
| **故事背景** | 世界观、时代、地点 |
| **主角** | 外貌、性格、服装（保持角色一致性关键） |
| **核心冲突** | 本集/本段的主要矛盾 |
| **情感基调** | 热血 / 温情 / 悬疑 / 喜剧 / 恐怖 等 |
| **美术风格** | 日漫 / 国漫 / 写实 / 赛博朋克 / 古风 等 |

若信息不足，**只问一次**，列出所有缺失项。

---

## Step 2：镜头规划

读取 `refs/shot-planning.md` 执行镜头规划。

规划输出格式（每集 6-12 个镜头）：

```
【镜头 N】
景别: 远景 / 全景 / 中景 / 近景 / 特写
角度: 平视 / 仰角 / 俯角 / 侧面
主体: 谁在画面中，做什么
背景: 场景描述
氛围: 光影、天气、情绪色彩
动态: 视频阶段的动作描述（单一动作，幅度适中）
台词/字幕: （可选）
```

将完整镜头表呈现给用户确认，等待用户批准后再进入下一步。

---

## Step 3：分镜图生成（make-image）

逐镜头执行，读取 `refs/prompt-templates.md` 构建提示词。

**关键规则：**
- 宽高比统一使用 `9:16`（漫剧竖屏格式）
- 每个提示词必须包含角色描述（保持一致性）
- 角色描述要在每个镜头中重复核心外貌特征

调用 neta 的 `make-image` 接口，参数：`-p "<提示词>" -a "9:16"`

从命令输出的 JSON 中提取图片 URL，记录到镜头表中：
```
镜头 N → 图片 URL: <url>
```

所有镜头图片生成完毕后，展示完整的图片 URL 列表，询问用户是否继续生成视频。

---

## Step 4：视频生成（make-video，model_w = wan2.6）

逐镜头执行图转视频。

**模型说明：**
- `model_w` = **wan2.6**，阿里巴巴通义实验室出品
- 支持 1080p / 24fps / 原生音画同步
- 支持多镜头叙事、口型对齐、角色声音一致性
- 单片段最长 15 秒

**提示词规则（严格执行）：**
- 提示词用于描述动效，如动作，镜头运动方式，光线变化，环境动画等
- 不要描述原图中已有的内容

调用 neta 的 `make-video` 接口，参数：`-i "<图片URL>" -p "<动态描述>" -m "model_w"`

从命令输出中提取视频 URL 或 artifact UUID，记录到镜头表：
```
镜头 N → 视频: <url/uuid>
```

**并发建议：** 测试单个镜头效果后再批量执行，避免队列拥堵。

---

## Step 5：视频合并（merge-video）

所有镜头视频生成完毕后，执行合并。

**合并指令格式：**
```
按顺序合并：镜头1视频URL, 镜头2视频URL, ..., 镜头N视频URL
```

调用 neta 的 `merge-video` 接口，按顺序传入所有视频 URL。

---

## Step 6：交付与总结

输出以下内容：

```
【漫剧成片】
标题: <剧目名称>
集数: 第 X 集
镜头数: N 个
总时长: 约 N 秒

成片 URL: <merge-video 输出的 URL>

镜头对照表:
镜头1 [景别] → 图片URL | 视频URL
镜头2 [景别] → 图片URL | 视频URL
...
```

询问用户是否需要：
- 调整某个镜头重新生成
- 制作下一集
- 添加配音/BGM（通过 make-song 生成）

## 快速参考

| 步骤 | 命令 | 关键参数 |
|------|------|----------|
| 生成分镜图 | `make-image` | `-a "9:16"` |
| 生成视频 | `make-video` | `-m "model_w"` |
| 合并成片 | `merge-video` | 按顺序传入所有视频 |
